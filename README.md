# xq-robot-docs

**以客户问题导向重构的机器人技术文档** - 让SLAM建图从技术细节转向用户体验

## 🎯 项目理念

传统技术文档：`从技术参数开始` → 我们的重构：`从用户目标开始`

**核心转变**：
- ❌ 旧模式："我要用Gmapping还是Cartographer？"
- ✅ **新模式："我要为机器人建一张环境地图"**

## 🔄 重构方法论

### 问题导向设计原则
1. **用户场景化** - 按"快速2D/沉浸式3D/专业大范围"划分，而非算法技术栈
2. **端到端工作流** - 四大步骤：选择→启动→遥控→保存，形成完整闭环
3. **技术透明化** - 系统自动匹配最佳算法，用户无需预先了解SLAM细节
4. **价值交付** - 最终交付"一张可用地图"，而非"学会使用工具"

### 具体重构效果
| 维度 | 传统文档 | 问题导向重构 |
|------|----------|--------------|
| **起点** | 技术参数 | 用户目标 |
| **流程** | 分散断裂 | 连贯工作流 |
| **决策** | 用户选择算法 | 场景自动匹配 |
| **价值** | 教会工具使用 | 交付可用成果 |

## 📚 文档结构

- `docs/slam/` - SLAM建图系列（当前PDF内容）
- `docs/navigation/` - 自主导航系列
- `docs/hardware/` - 硬件使用指南
- `templates/` - 文档重构模板

## 🛠️ 技术特色

- **一键启动** - 简化复杂的ROS命令链
- **统一交互** - 跨模式的标准操作流程
- **智能推荐** - 基于硬件的自动算法选择
- **故障自愈** - 内置常见问题解决方案



## 🚀 项目价值

**从"技术参数"到"用户目标"的机器人文档革命**

- 5分钟内完成机器人建图，无需ROS背景
- 端到端工作流，告别碎片化操作
- 智能场景匹配，自动推荐最佳方案
